# 简介
评估构建端到端机器学习解决方案包括：
* 设计可扩展的数据管道
  * 数据处理
    * 处理缺失或不一致的数据
    * 使用 Spark 或 Flink 等框架、分布式技术进行大规模数据处理
  * 特征工程
    * 数据归一化和缩放
    * 特征提取与选择
    * 自动化特征工程流程
* 选择合适的算法和模型
  * 模型设计和训练包括：
    * 选择合适的算法（例如，深度学习、梯度提升）
    * 超参数调优与验证
    * 针对大型数据集的分布式系统训练
    * 实施提前停止和正则化
* 数据质量处理（定期审核数据是否存在偏差、整合多种数据来源、在模型评估过程中使用公平性指标）和预处理
* 确保模型部署和监控
  * 跟踪准确率、延迟和吞吐量等指标
  * 检测模型漂移和数据分布变化
  * 根据需要自动重新训练和更新模型
  * 异常日志记录和警报系统
  * 模型版本控制和回滚策略
* 应对延迟（优化模型推理速度、使用硬件加速器例如 GPU、模型复杂性和速度之间的平衡）、可扩展性高效（分布式、缓存、批量处理）和可复现性（维护代码和数据版本控制、使用 MLflow 等实验跟踪工具、实现流程自动化以获得一致的结果）等挑战

## 约束 
通过机器学习来实现核心业务功能，需要：
* 定义机器学习目标 - 业务⽬标可能是将销售额提⾼ 20% 或改善⽤户留存率。但⽬标可能没有明确定义，不能仅仅通过告诉模型 “提⾼ 20% 的销售额” 来训练它。为了让机器学习系统解决任务，需要将业务⽬标转化为明确的机器学习⽬标。⼀个好的机器学习⽬标是机器学习模型能够解决的⽬标。
* 指定系统的输入和输出（比如输入文本还是图片，输出布尔值还是连续数值）
  * 分类模型 - 分类是预测离散类别标签的任务，例如预测输⼊图像是否应该被分类为狗或猫。分类模型可以分为 2 组：⼆分类模型、多分类模型。
  * 回归模型 - 回归是预测连续数值的任务，例如预测房价或股票价格。回归模型可以分为线性回归和非线性回归。回归的核心思想是拟合（Fitting），比如坐标轴上散落着很多点，回归模型就是尝试画出一条最能代表这些点趋势的线。线性回归（Linear Regression）：假设关系是一条直线。方程类似于 y = ax + b。比如：工龄越长，工资越高。非线性回归：关系可能是曲线（比如指数增长）。
* 选择正确的机器学习类别（保证机器学习模型与业务目标保持一致）
  * 监督学习 - 通过使⽤带标签的数据集来学习任务。实际上，许多问题都属于这⼀类别，因为从带标签的数据中学习通常能得到更好的结果。
  * 无监督学习 - 通过处理不含正确答案的数据来进⾏预测。⽆监督学习模型的⽬标是从数据中发现潜在结构。
  * 强化学习 - 计算机智能体通过与环境的反复试错互动来学习执⾏任务。例如，可以训练机器⼈在房间中⾏⾛，也可以通过强化学习让 AlphaGo 这样的程序与对⼿在围棋中竞技。

### 数据相关
1. 数据准备 - 数据源 -> 数据工程 -> 特征工程 -> 预备特征
2. 数据工程 - 是设计和构建⽤于收集、存储、检索和处理数据的管道的实践。
3. 数据来源 - 机器学习系统可以处理来⾃许多不同来源的数据。了解数据来源是回答许多背景问题的好⽅法，这些问题可能包括：谁收集的数据？数据有多⼲净？数据来源是否可信？数据是⽤户⽣成的还是系统⽣成的？
4. 数据存储 - 数据库，如何存储多模态数据
   1. ETL（提取、转换、加载）
   2. 数据类型 - 结构化数据、非结构化数据、半结构化数据
   3. 数值数据
   4. 类别数据
5. 特征⼯程 - 包括两个过程。选择合适的特征是开发和训练机器学习模型时最重要的决策之⼀。选择最有价值的特征⾄关重要，⽽这⼀特征⼯程过程需要领域专家的知识，同时也⾼度依赖于具体任务。
   1. 使⽤领域知识从原始数据中选择和提取具有预测能⼒的特征
   2. 将预测特征转换为模型可⽤的格式
6. 特征工程操作 - ⼀旦选择了预测特征，就需要通过特征⼯程操作将它们转换为合适的格式。通常，⼀些选定的特征并不是模型可以直接使⽤的格式。特征⼯程操作可以将这些特征转换为模型可使⽤的格式。技术包括处理缺失值、缩放值有偏分布的数据，以及编码类别特征。
   1. 处理缺失值 - 没有⼀种处理缺失值的⽅法是完美的，每种⽅法都有其权衡。
      1. 删除 - 删除可以分为⾏删除和列删除。在列删除中，如果某个特征的缺失值太多，可以移除整个表示该特征的列。在⾏删除中，如果某个数据点有许多缺失值可以移除这⼀行。
      2. 填补 - ⽤默认值、均值、中位数或众数（最常⻅的值）填补缺失值。
   2. 特征缩放 - 是指将特征缩放到标准范围和分布的过程。当数据集中的特征处于不同的范围时，许多机器学习模型在学习任务时会遇到困难。例如，年龄和收⼊这样的特征可能有不同的取值范围。此外，当⼀个特征具有偏态分布时，⼀些模型在学习任务时也可能遇到困难。
      1. 归⼀化（最⼩-最⼤缩放）- 在这种⽅法中，特征被缩放到所有值都在［0，1］的范围内。归⼀化不会改变特征的分布，如果要将特征的分布更改为标准分布，则使⽤标准化。
      2. 标准化（Z-score 归⼀化）- 标准化是将特征的分布更改为均值 0，标准差为 1 的过程。
      3. 对数缩放 - 为了减⼩特征的偏度，可以使⽤⼀种称为对数缩放的常⽤技术。
   3. 离散化（分箱）- 离散化是将连续特征转换为类别特征的过程。
   4. 码类别特征 - 在⼤多数机器学习模型中，所有的输入和输出都必须是数值型的。这意味着如果某个特征是类别型的，应该在将其传递给模型之前将其编码为数值。将类别特征转换为数值表示的三种常⻅⽅法是整数编码、独热编码和嵌⼈学习。
      1. 紧数编码 - 为每个唯⼀的类别值分配⼀个整数值。
      2. 独热编码 - 会为每个唯⼀值创建⼀个新的⼆进制特征。
      3. 嵌入学习 - 是⼀种将类别特征映射到 N 维向量的⽅法。当特征可能取的唯⼀值数量⾮常⼤时，这种⽅法很有⽤。在这种悄况下，独热编码不是⼀个好的选择，因为它会导致⾮常⼤的向量⼤⼩。

## 模型开发
指选择合适的机器学习模型并对其进行训练以解决当前任务的过程。

### 模型选择
模型选择过程如下：
1. 建⽴⼀个简单的基线 - 例如，在视频推荐系统中，可以通过推荐最受欢迎的视频来获得基线。
2. 尝试简单的模型 - 在有了基线之后，探索⼀些训练速度较快的机器学习算法是⼀个好的做法，例如逻辑回归。
3. 切换到更复杂的模型 - 如果简单模型⽆法提供令⼈满意的结果，那么可以考虑更复杂的模型，如深度神经⽹络。
4. 如果需要更精确的预测，可以使⽤模型集成 - 使⽤多个模型的集成⽽不是单⼀模型可能会提⾼预测质量。创建模型集成有三种⽅式：bagging、boosting 和 stacking

**⼀些常⻅的模型选项包括：**
* 逻辑回归
* 线性回归
* 决策树
* 梯度提升决策树和随机森林
* 支持向量机
* 朴素⻉叶斯
* 因⼦分解机（FM）
* 神经⽹络

在选择机器学习算法时，重要的是考虑模型的不同⽅面。例如：模型需要⽤于训练的数据量；训练速度；超参数选择和超参数调优技术；持续学习的可能性；计算需求，⼀个更复杂的模型可能提供更⾼的准确性，但可能需要更多的计算能⼒，如 GPU ⽽⾮ CPU；模型的可解释性，⼀个更复杂的模型可能提供更好的性能，但其结果可能更难以解释。  

### 模型训练
⼀旦完成了模型选择，就可以开始训练模型了。在这个步骤中需要考虑：
* 构建数据集，有 5 个步骤
  1. 收集原始数据
  2. 识别特征和标签 - 获取标签有两种常⻅⽅式：⼈⼯标注和⾃然标注。⼈⼯标注。这意味着个别标注者⼿动标注数据。例如，个别标注者标注⼀个帖⼦是否包含虚假信息。由于有⼈⼯参与，⼈⼯标注通常能得到准确的标签。然⽽，⼈⼯标注也有很多缺点；它成本⾼且速度慢，会引⼊偏差，需要领域知识，并且对数据隐私构成威胁。⾃然标注。在⾃然标注中，真实标签是⾃动推断的，⽆需⼈⼯标注。通过⼀个例⼦更好地理解⾃然标签。假设想设计⼀个基于相关性对新闻推送进⾏排序的机器学习系统。解决这⼀任务的⼀个可能⽅法是训练⼀个模型，该模型以⽤户和帖⼦为输⼊，并输出⽤户在看到该帖⼦后按下点赞按钮的概率。在这种情况下，训练数据是⽤户，帖⼦对及其对应的标签，如果⽤户点赞该帖⼦则标签为 1，否则为 0。通过这种⽅式，能够⾃然地对训练数据进⾏标注，⽽⽆需依赖⼈⼯标注。
  3. 选择采样策略 - 通常，收集所有数据是不现实的，因此采样是减少系统数据量的⼀种有效⽅式。常⻅的采样策略包括便利采样、滚雪球采样、分层采样、⽔库采样和重要性采样。
  4. 划分数据 - 数据划分是指将数据集分为训练集、评估（验证）集和测试集的过程。
  5. 解决类别不平衡问题 - 具有偏斜类别标签的数据集被称为不平衡数据集。数据集中占较⼤⽐例的类别称为多数类，⽽占较⼩⽐例的类别称⼒少数类。平衡数据集是模型训练中的⼀个严重问题，因为这意味着模型可能没有⾜够的数据来学习少数类。有多种技术可以缓解这个问题。两种常⽤的⽅法：重新采样训练数据和更改损失函数。
     * 重新采样训练数据 - 重新采样是指调整不同类别之间的⽐例，使数据更加平衡。例如，可以对少数类进⾏过采样或对多数类进⾏⽋采样
     * 改损失函数 - 此技术通过更改损失函数，使其对类别不平衡更具鲁棒性。⾼层次的思路是对来⾃少数类的数据点赋予更⾼的权重。在损失函数中赋予更⾼的权重，可以在模型对少数类预测错误时对其进⾏更⼤的惩罚，从⽽迫使模型更有效地学习少数类。
* 选择损失函数 - 损失函数是衡量模型在预测预期结果时的准确性。损失函数使优化算法能够在训练过程中更新模型参数，以最小化损失。设计一种新的损失函数很困难，通常情况下是在已有的损失函数列表中根据具体情况选一个合适的，并按需进行一些小改动。
* 从头训练与微调 - 微调是指通过对模型的已学参数进行微小调整，继续在新数据上训练模型。
* 分布式训练 - 通过在多个工作节点之间分配任务，并使得工作节点并行操作，从而加快模型训练速度。主要有两种类型：数据并行和模型并行。

讨论要点
以下是⼀些讨论要点：
- 模型选择：哪些机器学习模型适合该任务，它们的优缺点是什么。在模型选择过程中可以考虑以下话题：
  - 训练所需时间、模型期望的训练数据量；模型可能需要的计算资源；模型在推理时的延迟；模型是否可以部署在⽤户设备上；模型的可解释性。使模型更复杂可能提⾼其性能，但结果可能更难解释；能否利⽤持续训练，还是需要从头开始训练；模型有多少参数？需要多少内存；对于神经⽹络，可能需要讨论典型架构/模块，如 ResNet 或基于 Transformer 的架构。还可以讨论超参数选择，如隐藏层数、神经元数量、激活函数等。
- 数据集标签：应该如何获取标签？数据是否经过标注，如果是，标注的质量如何？如果有⾃然标签，如何获取它们？如何接收⽤户对系统的反馈？获取⾃然标签需要多⻓时间？
- 模型训练。
  - 应该选择什么损失函数？（例如，交叉熵，MSE，MAE，Huber 损失等）
  - 应该使⽤什么正则化⽅法？（例如，L119，L219，熵正则化，K 折交叉验证，或 dropout）
  - 什么是反向传播？
  - 可能需要描述⼀些常⻅的优化⽅法，如 SGD、AdaGrad、Momentum 和 RMSProp。
  - 想使⽤哪些激活函数以及原因是什么？（例如，ELU、ReLU、Tanh、Sigmoid）。
  - 如何处理不平衡数据集？
  - 什么是偏差/⽅差权衡？
  - 过拟合和⽋拟合的可能原因是什么？如何解决它们？
- 持续学习：是否希望在每个新数据点上在线训练模型？是否需要将模型个性化到每个⽤户？多⻓时间重新训练⼀次模型？⼀些模型需要每天或每周重新训练，另⼀些则需要每⽉或每年。

## 评估
模型开发完成后的下⼀步是评估，这是使⽤不同的指标来理解机器学习模型性能的过程。有两种评估⽅法：离线评估和在线评估。  

### 离线评估
离线评估是指在模型开发阶段评估机器学习模型的性能。为了评估模型，通常⾸先使⽤评估数据集进⾏预测。接下来，使⽤各种离线指标来衡量预测值与真实值的接近程度。下面展示了⼀些常⽤于不同任务的常用离线评估指标。  
Task | Offline metrics
--- | ---
Classification | Precision, recall, F1 score, accuracy, ROC-AUC, PR-AUC, confusion matrix
Regression | MSE, MAE, RMSE
Ranking | Precision@k, recall@k, MRR, mAP, nDCG
Image generation | FID, Inception score
Natural language processing | BLEU, METEOR, ROUGE, CIDEr, SPICE

### 在线评估
在线评估是指在模型部署后，评估其在生产环境中的表现。为了衡量模型的影响，需要定义不同的指标。在线指标是指在在线评估中使用的指标，通常与业务目标相关。下面展示了针对不同问题的在线评估的各种指标。  
Problem | Online metrics
--- | ---
Ad click prediction | Click-through rate, revenue lift, etc.
Harmful content detection | Prevalence, valid appeals, etc.
Video recommendation | Click-through rate. total watch time. number of completed videos, etc
Friend recommendation | Number of requests sent per day, number of requests accepted per day, etc.

## 部署与服务
为在线和离线评估选择合适的指标后的⾃然下⼀步是将模型部署到⽣产环境中，为数百万⽤户提供
服务。需要涵盖的⼀些重要主题包括：
- 云端部署与设备端部署
- 模型压縮
- ⽣产环境中的测试
- 预测流⽔线

### 云端与设备端部署的权衡
| Cloud | On-device
--- | --- | ---
Simplicity | Simple to deploy and manage using cloud-based services | Deploying models on a device is not straightforward
Cost | Cloud costs might be high | No cloud cost when computations are performed on-device
Network latency | Network latency is present | No network latency
Inference latency | Usually faster inference due to more powerful machines | ML models run slower
Hardware constraints | Fewer constraints | More constraints, such as limited memory, battery consumption, etc.
Privacy | Less privacy as user data is transferred to the cloud | More privacy since data never leaves the device
Dependency on internet connetion | Internet connection needed to send and data to the cloud | No internet connection needed

### 模型压縮
模型压缩是指使模型更⼩的过程。这对于减少推理延迟和模型⼤⼩是必要的。压缩模型的三种常⽤技术是：
- 知识蒸馏：知识蒸馏的⽬标是训练⼀个⼩模型（学⽣模型）来模仿⼀个⼤模型（教师模型）。
- 剪枝：剪枝是指找到最不重要的参数并将它们置零。这样可以得到更稀疏的模型，使存储更加⾼效。
- 量化：模型参数通常⽤32位浮点数表示。在量化过程中，我们使⽤更少的位数来表示参数，从⽽减⼩模型的⼤⼩。量化可以在训练过程中或训练后进⾏。

### ⽣产环境中的测试
确保模型在⽣产环境中表现良好的唯⼀⽅法是⽤真实流量进⾏测试。常⽤的模型測试技术包括影⼦部署、A/B 测试、⾦丝雀发布、交叉实验、多臂赌博机等。

### 预测流⽔线
要在⽣产中服务请求，需要⼀个预测流⽔线。
⼀个重要的设计决策是在线预测与批量预测之间批量预测。在批量预测中，模型会定期进⾏预测。由于预测是预先计算的，不需要担⼼模型⽣成预测时的时间问题。
在线预测。在在线预测中，预测会在请求到达时⽴即⽣成并返回。在线预测的主要问题是模型可能需要较⻓时间来⽣成预测。

机器学习系统在部署到⽣产环境后可能失败的原因有很多。最常⻅的原因之⼀是数据分布变化。数据分布变化是指模型在⽣产环境中遇到的数据与训练时遇到的数据不同的情况。
现实世界中的数据分布不断变化。换句话说，随着时间的推移，训练时使⽤的数据可能会变得不再相关。这会导致模型陈旧，性能随时间下降。因此，我们应该持续监控系统以检测这⼀问题。应对数据分布变化的两种常⻅⽅法是：
- 使⽤⼤型数据集进⾏训练。⾜够⼤的训练数据集可以使模型学习到全⾯的分布，因此⽣产环境中遇到的任何数据点很可能来⾃这个学习到的分布。
- 定期使⽤新分布中的有标签数据重新训练模型。

机器学习系统中的监控技术分为两类：与操作相关的指标和机器学习特定的指标。
与操作相关的指标：这些指标确保系统正常运⾏。它们包括平均服务时间、吞吐量、预测请求数量、CPU/GPU 利⽤率等。
机器学习特定的指标：
- 监控输⼊/输出。模型的表现取决于它所接收的数据，因此监控模型的输⼊和输出⾄关重要。
- 漂移。监控系统的输⼊和模型的输出，以检测它们底层分布的变化。
- 模型准确性。例如，期望准确性在特定范围内。
- 模型版本。监控部署的是哪个版本的模型。


# Visual Search System 视觉搜索系统  
该系统帮助⽤户发现与所选图像在视觉上相似的图像。设计⼀个类似于 Pinterest的 视觉搜索系统  
该系统根据⽤户提供的查询图像检索相似的图像，并根据它们与查询图像的相似度进⾏排序，然后显示给⽤户。平台仅⽀持图像，不允许视频或⽂本查询。为简化起⻅，不需要个性化。  

### 选择正确的机器学习类别
模型的输出是⼀组与查询图像相似的排序图像。因此，视觉搜索系统可以被框定为⼀个排序问题。通常，排序问题的⽬标是根据与查询的相关性对⼀组项⽬（如图像、⽹站、产品等）进⾏排序，使得更相关的项⽬在搜索结果中排名更⾼。许多机器学习应⽤，如推荐系统、搜索引擎、⽂档检索和在线⼴告，都可以被框定为排序问题。这里将使⽤⼀种⼴泛使⽤的⽅法，称为表示学习。  

在表示学习中，模型被训练将输⼊数据（如图像）转换称为嵌⼈的表示。另⼀种描述⽅式是，模型将输⼊图像映射到称为嵌⼈空间的 N 维空间中的点。学习这些嵌⼈，使得相似的图像在空间中具有彼此接近的嵌⼊。下图展示了两个相似的图像如何被映射到嵌⼊空间中的两个接近的点。为演示起⻅，在⼀个⼆维空间中可视化图像嵌⼈（以 ’xxx’ 表示）。实际上，这个空间是 N 维的，其中 N 是嵌⼈向量的⼤⼩。  
![](./image-embedded.png)  

如何使⽤表示学习对图像进⾏排序？  
⾸先，输⼊图像被转换嵌入向量。接下来，通过测量查询图像与平台上其他图像在嵌入空间中的距离来计算它们的相似度分数。图像根据相似度分数进行排序
   
## 数据准备
### 数据工程
除了通用的数据工程基础知识外，了解可用的数据也很重要。由于视觉搜索系统主要关注用户和图像，有以下可⽤数据：图像、⽤户、⽤户-图像交互  
图像创作者上传图像，系统存储这些图像及其元数据，如所有者ID、上下⽂信息（如上传时间）、标签等。用户⽤户数据包含与⽤户相关的⼈⼝统计属性，如年龄、性别等。⽤户-图像交互交互数据包含不同类型的⽤户交互。根据收集到的需求，主要的交互类型是展示和点击。  

### 特征⼯程
当使⽤了⼀个期望图像作输⼈的模型。图像在传递给模型之前需要进⾏预处理。来看⼀些常⻅的图像预处理操作：
- 调整⼤⼩：模型通常需要固定的图像尺⼨（例如，224×224）
- 缩放：将图像的像素值缩放到 0 和 1 的范围内
- Z-score 归⼀化：将像素值缩放为均值为 0，⽅差为 1
- 颜⾊致性：确保图像具有⼀致的颜⾊模式（例如，RGB 或 CMYK）

## 模型开发
### 模型选择
选择神经⽹络的原因是：
- 神经⽹络擅⻓处理⾮结构化数据，例如图像和⽂本。
- 与许多传统机器学习模型不同，神经⽹络能够⽣成在表示学习中所需的嵌⼈。

应该使⽤哪种神经⽹络架构？架构必须适⽤于图像。基于 CNN 的架构，如 ResNet，或者更新的基于 Transformer 的架构，如 ViT，在处理图像输⼈时表现良好。下图展示了⼀个将输⼈图像转换嵌⼊向量的简化模型架构。卷积层的数量、全连接层中的神经元数量以及嵌⼊向量的⼤⼩是通常通过实验选择的超参数。

### 模型训练
⼀种常⻅的学习图像表示的技术是对⽐训练。使⽤这种技术，训练模型以区分相似和不相似的图像。为模型提供⼀个查询图像，⼀个与查询图像相似的图像，以及⼀些不相似的图像。在训练过程中，模型学习⽣成的表示使得相似图像⽐其他图像更接近查询图像。要使⽤对⽐训练技术训练模型，⾸先需要构建训练数据。

### 构建数据集
如前所述，每个⽤于训练的数据点包含⼀个查询图像、⼀个与查询图像相似的正样本图像，以及 n-1 个与查询图像不相似的负样本图像。数据点的真实标签是正样本图像的索引。如下图所示，除了查询图像（q）之外，还有 n 个其他图像，其中⼀个与 q 相似（狗的图像），其他 n-1 个图像是不相似的。该数据点的真实标签是正样本图像的索引，即 2（下图中 n 个图像中的第⼆张图像）。  
![](./训练数据点.png)  

为了构建⼀个训练数据点，随机选择个查询图像和 —1 个负样本图像。选择正样本图像时，有以下三种选择：
- 使⽤⼈⼯判断 - 这种⽅法依赖⼈⼯标注者⼿动查找相似图像。⼈⼯参与能够产⽣准确的训练数据，但使⽤⼈⼯标注成本⾼且耗时。
- 使⽤⽤户点击等交互数据作为相似性的代理 - 这种⽅法中，基于交互数据来衡量相似性。例如，当⽤户点击某个图像时，点击的图像被视⼒与查询图像 q 相似。这种⽅法不需要⼈⼯操作，可以⾃动⽣成训练数据。然⽽，点击信号通常噪声很⼤。⽤户有时即使在图像与查询图像不相似时也会点击。此外，这种数据⾮常稀疏，对于很多图像我们可能没有点击数据。使⽤噪声⼤且稀疏的训练数据会导致较差的性能。
- ⼈⼯⽣成与查询图像相似的图像，即⾃监督 - 这种⽅法中，⼈⼯⽣成⼀个与查询图像相似的图像。例如，可以通过旋转查询图像进⾏增强，并将新⽣成的图像作为相似图像。最近开发的框架，如 SimCLR 和 MoCo，采⽤了相同的⽅法。种⽅法的优势在于不需要⼈⼯操作。可以通过简单的数据增强逻辑来创建相似图像。此外，构建的训练数据没有噪声，因为增强图像总是与原始图像相似。这种⽅法的主要缺点是构建的训练数据与真实数据不同。在实际操作中，相似图像并不是查询图像的增强版本；它们在视觉和语义上相似，但去是不同的。

一旦构建了数据集，就可以使⽤合适的损失函数训练模型。  

### 选择损失函数
如下图所示，模型将图像作为输⼈，并为每个输⼊图像⽣成⼀个嵌⼈。Ex 表示图像 x 的嵌入。训练的⽬标是优化模型参数，使相似图像在嵌⼊空间中彼此靠近。如下图所示，正样本图像和查询图像在训练过程中应变得更加接近。为了实现这⼀⽬标，需要使⽤损失函数来衡量⽣成的嵌⼈的质量。不同的损失函数是为对⽐训练设计的，了解对⽐损失函数的⼯作原理是很重要的。  
![](./嵌入模型.png)  


- 计算相似度。⾸先，我们计算查询图像与其他图像嵌⼊之间的相似度。点积和余弦相似度⼴泛⽤于测量嵌⼈空间中点之间的相似度。欧⼏⾥得距离也可以⽤来测量相似度。然⽽，欧⼏⾥得距离在⾼维空间中通常表现较差，这是由于维度灾难的影响。
- Softmax。在计算的距离上应⽤ Softmax 函数。这确保了值的总和为 1，从⽽可以将这些值解释为概率。
- 交叉熵。交叉熵衡量预测概率与真实标签的接近程度。当预测概率接近真实标签时，说明嵌⼈⾜以区分正样本图像和负样本图像。
  
还可以讨论使⽤预训练模型的可能性。例如，可以利⽤⼀个预训练的对⽐模型，并使⽤训练数据对其进⾏微调。这些预训练模型已经在⼤型数据集上进⾏了训练，因此它们学到了良好的图像表示。与从头开始训练模型相⽐，这可以显著减少训练时间。

## 评估
在开发完模型后，可以讨论评估。

### 离线指标
根据给定的需求，评估数据集可⽤于离线评估。假设每个数据点都有⼀个查询图像、⼀些候选图像，以及每个候选图像和查询图像对的相似度得分。相似度得分是⼀个介于 0 到 5 之间的整数，其中 0 表示没有相似性，5 表示两个图像在视觉上和语义上⾮常相似。对于评估数据集中的每个数据点，将模型⽣成的排序与基于真实得分的理想排序进⾏⽐较。  

来看看搜索系统中常⽤的离线指标。注意，搜索、信息检索和推荐系统通常共享相同的离线指标。
- 平均倒数排名（MRR）- 指标通过考虑模型⽣成的每个输出列表中第⼀个相关项的排名，并对它们进⾏平均来衡量模型的质量。
- 召回率@k - 该指标衡量输出列表中相关项的数量与整个数据集中可⽤的相关项总数之间的⽐率。
- 准确率@k
- 平均准确率（mAP）
- 归⼀化折扣累积增益（nDCG）
