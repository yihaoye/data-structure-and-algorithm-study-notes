# 性能工程
性能工程（Performance Engineering）是优化 IT 系统以满足速度与效率基准的实践。性能工程并非单一操作，而是一种 DevOps 与左移方法论，支持企业在软件开发生命周期 (SDLC) 的每个环节跟踪并优化性能。其目标是确保系统满足速度、可靠性、效率、响应时间等维度的性能指标。

### 归纳：实践工具列表
* 线上出问题时能不能快速定位根因的核心能力，尤其是处理 "系统在生产环境行为异常" 的，而这类问题往往不是应用代码本身的 bug，而是资源、网络、内核层面的东西在作怪
  * 熟练命令行操作、shell 脚本、常用工具链（grep/awk/sed/curl/netstat/lsof 这些排障利器）
  * 理解进程、文件描述符、内存/CPU/磁盘 I/O 这些基础资源模型，能看懂 top、vmstat、iostat 的输出并知道意味着什么
  * 网络基础：TCP/IP、DNS、路由，能用 tcpdump、ss 之类的工具排查网络问题
  * 会看日志、会用 strace/perf 定位一个服务为什么突然变慢或者挂了
* 特定工具的详细参数，比如某个 perf 子命令的所有 flag — 这些完全可以现查现用，记住 "有这么个工具能干什么事" 就够，具体参数交给搜索引擎或 AI，可以跳过或粗看
