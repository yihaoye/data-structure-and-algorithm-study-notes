[停机问题（halting problem）](https://zh.wikipedia.org/zh-hans/%E5%81%9C%E6%9C%BA%E9%97%AE%E9%A2%98)是逻辑数学中可计算性理论的一个问题。就是判断任意一个程序是否能在有限的时间之内结束运行的问题。该问题等价于如下的判定问题：是否存在一个程序 P，对于任意输入的程序 w，能够判断 w 会在有限时间内结束或者死循环。  

艾伦·图灵用对角论证法证明了，不存在解决停机问题的通用算法。或者说不存在通用算法，能判断任意状态机（程序即是状态机，是图灵完备的系统：有条件分支 + 任意循环，理论上等价于图灵机）在任意输入下是否停机 — 不是每个具体案例都无法判断，而是无法造出一个万能的判断器。  
这个证明的关键在于对计算机和程序的数学定义，这被称为图灵机。停机问题在图灵机上是不可判定问题。这是最早提出的决定性问题之一。停机问题的证明意义超出计算机科学本身，它回答了一个根本性的问题：计算的边界在哪里？  
在图灵之前，数学家们普遍相信希尔伯特的梦想：即任何数学问题，原则上都能被机械地判定真假。图灵用停机问题的证明彻底打碎了这个梦想，证明了：存在计算机永远无法解决的问题，不是因为算力不够，而是在逻辑上就不可解。  
  
## 证明过程
除了对角论证法，一个常见的停机问题证明是通过反证法解决的。假设存在一个程序 H，能够判断任意程序 P 和输入 I 是否会停机。True 表示判定会停机，False 表示判定会死循环。  
首先，程序本身可以作为输入。那么现在构造一个悖论程序 D，它接受一个程序 P 作为输入，并使用 H 来判断 P 是否在输入 P 上停机。D 的定义如下（简单地说就是总是会反向操作）：  
```python
def D(P):
    if H(P, P) == True: # 如果 P 在输入 P 上停机
        while True: # 执行无限循环
            pass
    else: # 如果 P 在输入 P 上死循环
        return # 执行停机
```  
现在，考虑 D(D) 的行为。根据 D 的定义，则有：
- 如果 H(D, D) 返回 true，意味着 D 在输入 D 上停机，那么 D(D) 将进入无限循环，这与 H 的判断矛盾。
- 如果 H(D, D) 返回 false，意味着 D 在输入 D 上死循环，那么 D(D) 将停机，这同样与 H 的判断矛盾。
  
因此，无论 H(D, D) 返回 true 还是 false，都会导致矛盾。这表明最初的假设是错误的，即不存在一个程序 H 能够判断任意程序 P 和输入 I 是否会停机。  
  
## 其他
一个对象比如一段文字的[柯氏复杂性](https://zh.wikipedia.org/zh-hans/%E6%9F%AF%E6%B0%8F%E5%A4%8D%E6%9D%82%E6%80%A7)（亦作描述复杂性、随机复杂度，或算法熵）是衡量描述这个对象所需要的信息量的一个尺度。  
与康托尔的对角论证法、哥德尔不完备定理和图灵的停机问题类似，柯氏复杂度的概念可以用于阐述和证明不可能性。  

### 停机问题与系统设计
状态机是有环有向图（DG）的紧凑表达，允许回头和循环，更通用，DAG（无环有向图）是它的子集，是一种常用的结构，也常用于描述任务依赖关系，结构静态确定、保证了没有循环，一定停机，从而避免了停机问题。  
在系统设计中，尤其是涉及到工作流或 pipeline 的设计时，停机问题是一个重要的考虑因素。比如在设计一个自动化的任务调度系统时，如果允许用户定义任意的任务依赖关系（条件路径）和循环，即工作流为状态机，那么就可能出现死循环的情况且无法判断、检测出来（停机问题），从而会导致系统无法正常运行、无法完成任务。为了解决这个问题，工作流或 pipeline 框架、库设计者通常会引入一些限制，以确保系统不会无限运行：  
* 主动限制表达能力让系统不再图灵完备，例如强制使用 DAG 结构（通过启动时或提交时运行拓扑排序检测保证）、或设置最大循环次数（有界循环）、或限制条件表达式。
* 又或在不限制表达能力的情况下，通过超时机制在运行时强制中断作为兜底。
